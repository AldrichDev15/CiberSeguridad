El **recorrido de directorios o rutas** es una vulnerabilidad que permite a un atacante leer archivos arbitrarios en el servidor que levanta una aplicación. Esto puede incluir código, datos, credenciales para sistemas en el backend, ficheros confidenciales del OS...
En algunos casos, se puede llegar a escribir sobre estos ficheros permitiendo modificar el comportamiento de la aplicación o sus datos y en última instancia, tomando control total del propio servidor.
Por ejemplo, una aplicación que muestra imágenes usando la siguiente URL `<img src="/loadImage?filename=218.png">` Las imágenes están ubicadas en el disco en `/var/www/images/`. Para devolver la imagen, la aplicación añade el filename al directorio base y usa una API del sistema de archivos para leer el contenido del archivo, es decir, la aplicación lee desde `/var/www/images/218.png` Esta aplicación no tiene defensas contra este tipo de ataques, por tanto, si ponemos `https://insecure-website.com/loadImage?filename=../../../etc/passwd` en los sistemas operativos basados en Unix (o `https://insecure-website.com/loadImage?filename=..\..\..\windows\win.ini` para los Windows) veremos que este archivo contiene detalles de los usuarios registrados en el servidor, aunque podríamos sacar información de otros archivos con la misma técnica.

El **Control de acceso** consiste en la aplicación de restricciones de quién o qué está autorizado a realizar qué tipo de acciones o acceder a qué recursos. En el contexto de las aplicaciones web, el control de acceso depende de la autenticación y la gestión de la sesión (identifica que peticiones HTTP posteriores están hechas por un mismo usuario). Las decisiones de diseño del control de acceso están hechas por humanos, con lo que el potencial de fallo es mayor.
El escalado de privilegios vertical se da cuando un usuario gana acceso a una funcionalidad para la cual no tiene permisos.
